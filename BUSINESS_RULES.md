# 업무 규칙

## 품목 매칭

- 품목코드가 DB에 있으면 정확 일치로 처리한다.
- 코드가 없거나 미등록이면 표준 품목명과 정규화된 이름을 비교한다.
- 모델, 색상, 기본형/핸디형, 케이스 여부를 구분한다.
- 후보가 하나로 충분히 좁혀지지 않으면 자동 확정하지 않는다.
- 결과 상태는 `exact`, `similar`, `ambiguous`, `missing`, `manual`, `alias`를 사용한다.

## 세트상품

- 판매처 상품은 여러 실제 재고 품목으로 구성될 수 있다.
- 예: `QP1000C 화이트 / 샌드`는 본품과 해당 색상 실리콘케이스의 세트일 수 있다.
- 세트 구성은 `registered_products`와 `product_components`를 기준으로 분해한다.
- 입력 수량에 구성품 수량을 곱한 값이 실제 이동/출고 수량이다.

## 수기 추가 품목

- 셀메이트의 표준 열 뒤에 작업자가 품목을 수기로 추가할 수 있다.
- `재고매칭(1/2)옵션내용` 및 우측 추가 열을 함께 읽는다.
- 수기 입력 품목은 누락하지 않되 기본적으로 유사 또는 확인필요 대상으로 검토한다.

## 합포장과 중복 출고

- 수령자 이름만 같다는 이유로 중복으로 판정하지 않는다.
- 주문번호·수령자·전화번호·우편번호·주소·상품·옵션·수량을 종합해 중복 키를 만든다.
- 같은 주문번호와 배송정보라도 서로 다른 상품 행은 정상적인 합포장이다.
- 동일 주문의 동일 상품 행이 반복되거나 이전 출고 이력에 존재할 때만 중복 출고로 표시한다.

## 오류 항목

- 빨간색 또는 확인필요 항목이 있어도 사용자가 확인 후 출고 Excel을 생성할 수 있다.
- 이카운트 실제 등록은 적용 품목코드, 품목명, 양수 수량이 모두 있어야 한다.
- 사용자가 수동으로 수정한 값은 화면에 즉시 반영한다.
- 관리자만 반복 사용을 위한 DB 별칭을 저장할 수 있다.

## 판매전표 금액과 배송비

- 스마트스토어 상품금액은 `최종 상품별 총 주문금액`을 최우선 기준으로 사용한다.
- 원본 상품금액과 실제 배송비의 합계가 전표 총금액과 1원이라도 다르면 저장하지 않는다.
- 미등록·확인필요 상품은 조용히 제외하지 않고, DB 추가 또는 분석 제외를 사용자가 명시적으로 처리해야 한다.
- 배송비는 `배송비 묶음번호`를 우선 고유키로 사용하고, 없으면 주문번호를 사용하여 한 번만 계산한다.
- 실제 배송비는 `배송비 합계 + 제주/도서 추가배송비 - 배송비 할인액`으로 계산한다.
- 추가배송비나 할인액이 있는 배송비 묶음은 화면에서 별도 색상으로 표시하여 사용자가 확인한다.
