# 변경 기록

## 1.0.16

- REQM 공식 브랜드 스토어의 검은 로고 타이포와 넓은 흰 여백을 프로그램 디자인에 반영
- 제품 카드 느낌의 밝은 회색·큰 둥근 모서리 카드 디자인 적용
- 공식 홈페이지 여름 이미지에서 가져온 민트·청록과 모래색을 상태·파일·출고지 포인트로 적용
- 로그인·파일 선택·출고 변환 주요 버튼을 REQM 로고와 같은 검은색으로 통일
- 표와 스크롤바를 브랜드 팔레트에 맞춰 밝고 단정한 스타일로 변경

## 1.0.15

- 파일 선택 버튼을 드래그 앤 드롭 영역 위로 이동하고 글자 크기에 맞게 축소
- 드래그 앤 드롭 영역을 중앙 정사각형 형태로 축소
- 로그인 입력 카드와 이메일·비밀번호 입력칸 너비 축소
- 면세점 출고지 선택창을 대표 출고지명이 보이는 길이로 축소하고 중복 채널명 제거
- 버전을 제목 오른쪽 아래의 작은 글자로 표시하고 별도 테두리 없이 정리

## 1.0.14

- 검증되지 않은 이카운트 창고이동 기능과 화면 버튼 제거
- 창고이동 API, 담당자·창고 조회, API 인증키 저장 및 전용 품목 기준표 코드 제거
- 기존 PC에 남아 있는 창고이동 API 인증키 파일을 업데이트 후 자동 삭제
- 로그인 시 불필요한 담당자·창고 테이블 조회를 제거해 초기 데이터 로딩 최적화

## 1.0.13

- 이카운트 API 직접실행과 동일한 `sboapi` 서버를 창고이동 설정에서 선택 가능
- 로그인과 창고이동 등록이 반드시 같은 API 서버를 사용하도록 통일
- 운영 `oapi` 서버 선택 기능을 유지하고 직접실행 호환 서버를 기본값으로 적용
- `SESSION_ID` URL 인코딩 및 `Accept`, `Content-Type`, `User-Agent` 요청 헤더 보완

## 1.0.12

- 이카운트 창고이동 API가 오류 응답을 반환할 때 `성공 0건 / 실패 0건`으로만 표시하던 문제 수정
- 중첩된 `Data`, `Error`, `Errors`, `Message`, `ResultDetails` 응답을 분석해 실제 이카운트 오류 문구 표시
- 이카운트 응답 형식이 예상과 다를 때 상태와 원본 응답을 표시해 등록 실패 원인을 즉시 확인 가능

## 1.0.11

- 이카운트 재고현황에서 받은 품목코드·품명 기준표 169개를 창고이동 전용 기준으로 내장
- 창고이동 입력 품목명을 이카운트 품명과 정확·유사 대조하여 실제 이카운트 품목코드 자동 적용
- `QP1000C`와 `QP1000C1`처럼 판매처·이카운트 간 모델 표기가 조금 다른 경우에도 제품 종류와 색상을 함께 비교
- Supabase 출고용 품목코드와 이카운트 창고이동용 품목코드를 분리해 최종 전송 코드 정확성 개선
- 실제 등록 확인창에서 최종 이카운트 코드·품명·수량을 표시

## 1.0.10

- 창고이동 등록 직전에 품목코드가 비어 있거나 DB에 없는 경우 품목명으로 표준 품목코드 자동 보완
- 세트상품 품목명은 DB 구성품 코드와 수량으로 자동 분해
- 품목명 후보가 여러 개이거나 DB에서 찾지 못하면 실제 API 전송을 차단하고 해당 행 안내
- 등록된 품목코드는 DB의 표준 품목명과 함께 이카운트 창고이동 요청에 사용

## 1.0.9

- 트래블메이트 매장 발주서 전용 구조 자동 인식 추가
- 상단 사업자 정보가 아니라 `TM 상품코드`, `TM 상품명`, `발주수량`, `매장명` 표를 출고 데이터로 분석
- 발주수량이 입력된 품목만 가져오고 판매처 상품코드와 품목명을 DB에 순차 대조
- 하단 매장 정보에서 출고 주소, 담당 매니저 및 연락처를 자동 추출
- 파일 내 출고지가 확인되면 별도의 출고지 선택 없이 매장 출고용 변환 활성화

## 1.0.8

- 출고 작업 파일 선택 버튼을 유지하면서 Excel·CSV·PDF 드래그 앤 드롭 영역 추가
- 텍스트·표 기반 PDF에서 주문번호, 수령인, 품목명, 품목코드, 수량, 주소, 연락처 등 출고 열 자동 탐지
- 파일의 품목코드를 DB 품목코드와 우선 대조하고 원본 품목코드를 분석 결과 표에 표시
- PDF에 주문번호 또는 수령인이 누락되면 오류 항목으로 표시하고 잘못된 출고 변환 방지
- 이미지로만 구성된 PDF는 OCR 필요 여부를 안내하도록 오류 메시지 개선

## 1.0.7

- 이카운트 창고이동 담당자를 코드뿐 아니라 이름으로도 인식해 담당자 코드로 변환
- 보내는 창고와 받는 창고를 코드뿐 아니라 창고명으로도 인식해 창고 코드로 변환
- 띄어쓰기와 영문 코드 대소문자 차이를 허용하고, 중복 이름은 목록 선택을 요구하도록 안전성 개선
- 등록되지 않은 이름을 API 코드로 그대로 전송하지 않도록 검증 강화

## 1.0.6

- 이카운트 API 인증키를 Windows DPAPI로 현재 PC·Windows 사용자에 암호화 저장
- 일반 사용자는 저장 여부와 마스킹 값만 확인하고 인증키 원문을 표시·수정할 수 없도록 제한
- 관리자만 인증키 표시, 변경, 암호화 저장 및 저장값 삭제 가능
- API 인증키 원문이 설정 파일·Supabase·GitHub·로그에 기록되지 않도록 분리

## 1.0.5

- 기존 B2C/B2B, 판매처 직접양식, 출고지, DB 관리 및 자동 업데이트 기능과 1.0.4 이카운트 창고이동 보드 통합
- 업데이트 시 실행 파일 잠금 재시도, 교체 파일 해시 검증 및 PyInstaller 재실행 환경 초기화 적용
- PDF 분석, 변환 참고파일 및 Windows VC 런타임을 함께 포함하는 통합 PyInstaller 빌드 설정 추가
- 소스에 입력되어 있던 기본 로그인 이메일 제거

## 1.0.4

- 이카운트 창고이동 보드 추가
- 담당자 및 창고 Supabase 조회와 코드 직접 입력 추가
- 달력 방식 일자 입력 추가
- Excel/PDF 드래그 앤 드롭과 파일 선택 추가
- 품목명/품목코드 분석 및 수동 수정 추가
- 이카운트 `SaveLocationTran` 로그인·등록·결과 확인 추가
- API 정보 입력을 우측 상단 팝업으로 이동
- PDF 빌드 의존성과 휴대 가능한 PyInstaller spec 추가
- 프로젝트 인수인계 문서 추가

## 기존 주요 기능

- 셀메이트 B2C 및 면세점 B2B 파일 처리
- 입력 양식 자동 판별
- Supabase 기반 단품/세트상품 매칭
- 유사·확인필요·미등록 표시 및 수동 수정
- 관리자 전용 품목 DB 관리
- 합포장과 중복 출고 구분
- 택배 출고용 Excel 생성
- AI 채팅 기능 제거
